# PLAN — Backend Endpoint Inventory (105 routes)

> Part of [PLAN.md](PLAN.md) Section 3. `dashboard/server.py` (ThreadingHTTPServer :8080). Generated 2026-06-06.

dashboard/server.py is a custom http.server BaseHTTPRequestHandler (threaded ThreadingHTTPServer) on :8080, NOT a framework. Routing is a single if/elif path-dispatch chain inside do_POST (server.py:3234), do_GET (server.py:4099), and do_OPTIONS (server.py:3227). path = urlparse(self.path).path (GET also strips trailing slash: rstrip('/') or '/'). There is no route table/decorator — every endpoint is a literal `if path == ...` / `elif path == ...`, with a few `path.startswith(...)` branches for dynamic IDs (/api/findings/{id}, /api/targets/{domain}, /api/sessions/{id}, /api/session/{id}, /api/recon/pipeline/{job_id}, /api/reports/{filename}, /api/codefix/status/{job_id}). Total: ~25 POST routes + ~70 GET routes (many GET are legacy aliases) plus OPTIONS catch-all.

SECURITY/AUTH: There is NO authentication or session/token check on any endpoint. The only access controls are: (1) ALL POST requests pass through _enforce_same_origin_post() (server.py:3094) before dispatch — rejects cross-origin POSTs with 403, allowlisting localhost/127.0.0.1 on the bind port + ports 3000/3001/3002/3010 + VIPER_WEBAPP_ORIGINS env; missing-Origin requests are allowed (server binds 127.0.0.1 only). (2) The /ws WebSocket handshake (_handle_websocket, server.py:923) validates Origin against the same allowlist, 403 on mismatch. (3) The SSE stream (/api/stream, /api/events) has NO Origin check (CORS headers only). GET endpoints have no Origin enforcement at all — they rely solely on the 127.0.0.1 bind. _cors_headers() is applied to all responses.

PERSISTENCE LAYERS: (a) data/viper.db SQLite — tables: targets, findings, attacks/attack_history, waf_fingerprints, audit_log (the HackMode/swarm source of truth read by /api/hack/* and /api/graph*). (b) data/evograph.db SQLite — sessions, attack-evolution graph (EvoGraph). (c) state/*.json files — guidance.json, approval_response.json, qa_response.json (agent IPC control channels), chat_history.json, project_settings.json, viper_state.json, codefix_status.json. (d) reports/ directory — HTML/MD report files. (e) logs/ directory. (f) In-memory dicts (lost on restart): _active_scans, _recon_pipeline_jobs (also stores CTF jobs), _codefix_jobs, _TERMINAL_SESSIONS, _chat_history. (g) Plugin stores: CTFFeedbackStore (core.ctf_feedback), KnowledgeBase (core.knowledge_base), FailureAnalyzer, SettingsManager. (h) Subprocesses: /api/hack/start and /api/recon/pipeline/start fork separate Python processes.

REAL-TIME CHANNELS: WebSocket at GET /ws (typed frames {type,payload}, ping keepalive, _ws_broadcast() pushes events, client list _ws_clients). SSE at GET /api/stream and GET /api/events (event_bus subscribe, 30s keepalive). Both fed by a shared event_bus + _ws_broadcast. Background scan/recon/ctf threads emit scan_started/phase_update/finding_new/log_line/scan_completed/scan_error/recon_pipeline_started/completed/error/ctf_completed events.

NOTABLE SECURITY ITEMS: (1) /api/terminal/execute is a sandboxed shell — !connect now requires an RFC1918 private IP (loopback/link-local/public/hostnames blocked; SSRF/DNS-rebinding defense, fix #6 at server.py:3447). (2) /api/terminal/connect (server.py:3509) does NOT apply that private-IP restriction — it registers any user@host:port SSH proxy session. (3) /api/hack/hunts, /api/hack/hunt, /api/hack/audit accept a ?db_path query param defaulting to data/viper.db, letting a caller point at an arbitrary SQLite path. (4) /api/settings GET returns the full raw settings dict (potential sensitive config exposure), no auth. (5) Path-traversal guards exist on /api/reports/{filename} (Path.name), /api/reports/delete (Path.name + '..' check), and _serve_static (resolved-prefix check).
